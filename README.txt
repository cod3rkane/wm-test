# Walmart Front-End Recruitment Test

PT-BR

## Requisitos

* Crie um layout de página de produto semelhante às imagens fornecidas em `/ resources`.
* Exibir uma lista de produtos usando o JSON estático fornecido em `/ public / data`.
* O card do produto deve exibir o título do produto, imagem, preço de lista, preço de desconto, avaliações e prestações.
* A medida que o usuário passa o mouse sobre o card do produto, o card deve exibir a marca do produto e o botão de adicionar ao carrinho.
* O usuário deve ser capaz de adicionar e remover produtos ao carrinho.
* Fornecer instruções sobre como fazer o build e executar o projeto.

## Notas

* Sinta-se livre para usar qualquer gerenciador de pacotes, bundlers.
* ** Não use ** frameworks como Bootstrap ou Foundation, por exemplo.
* ** Não minifique ** qualquer um dos arquivos que foram gerados (CSS, JS).
* Criar um servidor estático para acessar os dados do JSON fornecido.

## Bônus

* Layout responsivo
* Unit Tests
* Manter a lista de produtos adicionados ao carrinho após recarregar a página
* Suporte para IE8+

## Avaliação

Avaliaremos o teste com base nos seguintes critérios:

- Estilo visual
- Limpeza, clareza e manutenibilidade do código
- Estrutura e organização do projeto
- Lógica de programação
- Resultado funcional

EN

## Requirements

* Create a product page layout similar to the images provided under `/resources`.
* Display a list of products using the static JSON provided under `/public/data`.
* The product card should display the product's title, image, list price, discount price, ratings and installments.
* As the user hovers over the product card, it should display the product's brand and the add to cart button.
* The user should be able to add and remove products to the cart.
* Provide instructions on how to build and run the project.

## Notes

* Feel free to use any package managers, bundlers.
* **Do not use** frameworks like Bootstrap or Foundation for example.
* **Do not minify** any of the files that were generated (CSS, JS).
* Create a static server in order to access the JSON data provided.

## Bonus

* Responsive layout
* Testes unitários
* Maintain the list of added products to the cart on page reload
* Support for IE8+

## Evaluation

We will evaluate the test based on the following criteria:

- Visual style
- Code cleanliness, clarity and maintainability
- Project structure and organization
- Programming logic
- Functional result
